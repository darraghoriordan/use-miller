# Design principles

- Build an application interface, not a starter-kit showcase.
- Prefer clear hierarchy, restrained colour, strong typography, and predictable spacing.
- Use accessible semantic elements and preserve visible keyboard focus.
- Design loading, empty, error, unauthorized, and offline states with the happy path.
- Keep marketing and authenticated application navigation conceptually separate.
- Use motion to clarify state changes and respect reduced-motion preferences.
- Make responsive layouts intentional at phone, tablet, and desktop widths.
- Keep user-facing terminology in product modules, not generic shared components.
